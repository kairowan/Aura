# Aura v1.0.0

Aura 首个正式桌面版本，完成了从开发态 Agent 工作台到可打包产品的第一轮落地。

## 项目规模

- 智能体：130 个
- 公共技能：218 个
- MCP 工具服务配置：129 个
- 内置核心工具：22 个

## 版本能力

- 桌面端：完成 macOS 桌面应用打包链，补齐运行时种子数据同步、前端 standalone 运行、启动错误兜底和启动性能优化。
- 多模型接入：支持自定义 AI 服务商配置，已补齐 Base URL、API Key、模型 ID 持久化与连接验证链路。
- 工作台体验：补齐智能体、技能、工具搜索；增加侧边栏 AI 服务商入口；优化设置弹窗和多个页面的 UI 可读性。
- 图表能力：集成 draw.io 生成与预览能力，支持在 Aura 对话中生成 `.drawio` 并在 artifact 面板预览。
- 自动化与渠道：新增自动化任务管理，以及飞书、Slack、Telegram 渠道接入页与热重载能力。
- 桌面自动化：提供截图、OCR、按文字点击、窗口内点击、滚动、拖拽、输入、快捷键、应用切换等桌面工具能力。
- 多平台准备：补齐 Windows/Linux 桌面打包配置和 CI；新增 Android/iOS 的 Capacitor 移动壳。

## 安装包与交付物

- macOS Apple Silicon：`Aura-1.0.0-arm64.dmg`
- Windows：`Aura-1.0.0-win.exe`
- Linux：`Aura-1.0.0.AppImage`、`Aura-1.0.0.deb`
- Android：`aura-mobile-android-debug.apk`
- iOS：仓库包含 `mobile/` 壳工程，可生成 Xcode 工程；设备安装包仍需要 Apple 签名流程

## 更新说明

- 修复桌面端打包后白屏、服务未就绪仍然跳转、前端资源缺失等稳定性问题。
- 修复 AI 服务商配置保存后未正确注入 `base_url` 的问题，并增加“测试连接”能力。
- 恢复并同步智能体、技能、工具配置到桌面运行时目录，避免首次安装后数据丢失。
- 新增 draw.io 图表生成与 artifact 预览。
- 新增自动化任务与多渠道接入。
- 新增桌面自动化与 OCR 能力。
- 新增移动端壳与多平台构建工作流。

## 已知限制

- 当前 macOS 安装包尚未完成签名与 notarization。
- Windows、Linux、Android 资产建议通过 GitHub Actions 在对应 runner 上生成并上传。
- iOS 真机安装包不在本次 release 资产内，需要后续接入 Apple Developer 签名与分发流程。
